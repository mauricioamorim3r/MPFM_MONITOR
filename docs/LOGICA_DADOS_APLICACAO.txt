3.1 METER (Medidor MPFM)
------------------------
interface Meter {
  id: string
  tag: string                    // "13FT0367"
  name: string                   // "Riser P5"
  location: 'TOPSIDE' | 'SUBSEA'
  status: 'active' | 'inactive'
  serialNumber: string
  manufacturer: string           // "Schlumberger"
  model: string                  // "Vx Spectra"
  lastCalibration: string        // "2026-01-18"
  kOil: number                   // 1.0908
  kGas: number                   // 1.09262
  kWater: number                 // 1.0
  daysToCalibration: number
}

Medidores FPSO Bacalhau:
  • 13FT0367 - Riser P5 (Topside)
  • 13FT0417 - Riser P6 (Topside)
  • 18FT1506 - PE_4 (Subsea)
  • 18FT1806 - PE_EO4 (Subsea)
  • 18FT1706 - PE_EO105 (Subsea)
  • 18FT1406 - PE_EO10 (Subsea)


3.2 MONITORING DATA ROW (Dados de Monitoramento Diário)
-------------------------------------------------------
interface MonitoringDataRow {
  id: string
  date: string
  
  // Dados Subsea
  subOil: number
  subGas: number
  subWater?: number
  subHC: number
  
  // Dados Topside
  topOil: number
  topGas: number
  topWater?: number
  topHC: number
  
  // Dados Separador
  sepOil: number
  sepGas: number
  sepWater?: number
  sepHC: number
  
  // Balanços Calculados
  hcBalTS: number                // % desvio HC Topside vs Separator
  totalBalTS: number             // % desvio Total
  hcStatus: 'OK' | 'ALERT' | 'FAIL'
  totalStatus?: 'OK' | 'ALERT' | 'FAIL'
  action: 'MONITORAR' | 'INVESTIGAR'
  
  // Dias consecutivos fora da tolerância
  subVsTS: number
  topVsTS: number
  topVsSub: number
}


3.3 CALIBRATION EVENT (Evento de Calibração)
--------------------------------------------
interface CalibrationEvent {
  id: string                     // "CAL-13FT0367-PER-20260118-001"
  meterId: string
  meterTag: string
  meterName: string
  type: 'Comissionamento' | 'Periódica' | 'Especial' | 'Investigação'
  status: 'Pendente' | 'Em Andamento' | 'Aguardando PVT' | 'Concluída'
  result?: 'Aprovado' | 'Reprovado'
  startDate: string
  endDate?: string
  currentStep: number            // 1-7 (Wizard de 7 etapas)
  progress: number               // 0-100%
  responsible: string
  createdAt: string
  updatedAt: string
}

Workflow de Calibração (7 Etapas):
  1. Registro Geral
  2. Dados PVT
  3. Totalizadores
  4. K-Factors
  5. Balanço de Massa
  6. Monitoramento Pós-Calibração
  7. Alarmes e Finalização


3.4 ALERT (Alerta do Sistema)
------------------------------
interface Alert {
  id: string
  type: 'error' | 'warning' | 'info' | 'success'
  severity: 'info' | 'warning' | 'critical'
  message: string
  meterId?: string
  meterTag?: string
  timestamp: string
  read: boolean
  actionRequired?: string
}

5. FLUXO DE PROCESSAMENTO COMPLETO
================================================================================

1. UPLOAD
---------
   Usuário faz upload de PDF/Excel via interface
   ↓

2. DETECÇÃO DE TIPO
-------------------
   detectPDFType() analisa conteúdo e nome do arquivo
   ↓

3. PARSING (Extração de Dados)
-------------------------------
   ┌─────────────────────────────────────────────────────────┐
   │ PDF B03    → parseB03Topside()                          │
   │              → { oil, gas, water, hc, total }           │
   ├─────────────────────────────────────────────────────────┤
   │ PDF B05    → parseB05Subsea()                           │
   │              → { oil, gas, water, hc, total }           │
   ├─────────────────────────────────────────────────────────┤
   │ Separator  → parseSeparator()                           │
   │              → { oil, gas, water, hc, total }           │
   ├─────────────────────────────────────────────────────────┤
   │ PVT        → parsePVTCalibration()                      │
   │              → { kFactors, composition, densities }     │
   ├─────────────────────────────────────────────────────────┤
   │ Excel      → parseExcelFile()                           │
   │              → { blocks: CUMULATIVE, DAY, AVERAGE }     │
   └─────────────────────────────────────────────────────────┘
   ↓

4. VALIDAÇÃO CRUZADA (crossValidator.ts)
-----------------------------------------
   Compara dados de múltiplas fontes:
   • Excel × PDF × Manual
   
   Classificação do desvio:
   ┌──────────────────┬────────────────────────────────┐
   │ CONSISTENTE      │ Todas fontes < 2% desvio       │
   │ ACEITAVEL        │ 2-5% desvio                    │
   │ INCONSISTENTE    │ > 5% desvio                    │
   │ FONTE_UNICA      │ Apenas 1 fonte disponível      │
   │ SEM_DADOS        │ Nenhum dado                    │
   └──────────────────┴────────────────────────────────┘
   ↓

5. ANÁLISE DIÁRIA (dailyAnalyzer.ts)
-------------------------------------
   Gera alertas baseados em regras:
   
   ┌─────────────────────────┬──────────────────────────────────┐
   │ BSW_HIGH                │ BSW > 0.5%                       │
   │ GAS_BALANCE_ERROR       │ Balanço de gás > 5%              │
   │ PRODUCTION_VARIATION    │ Variação produção > 20%          │
   │ MISSING_DATA            │ Dados faltantes                  │
   └─────────────────────────┴──────────────────────────────────┘
   
   Análise de tendência:
   • Média móvel 7 dias
   • Desvio padrão
   • Coeficiente de variação
   ↓

6. CÁLCULO DE BALANÇO DE MASSA
-------------------------------
   Fórmulas:
   
   HC Balance (%) = ((Topside_HC - Subsea_HC) / Subsea_HC) × 100
   Total Balance (%) = ((Topside_Total - Separator_Total) / Separator_Total) × 100
   
   Status:
   ┌─────────────────┬─────────┬──────────────┐
   │ Desvio          │ Status  │ Cor          │
   ├─────────────────┼─────────┼──────────────┤
   │ HC > 10%        │ FAIL    │ Vermelho     │
   │ HC > 7%         │ ALERT   │ Amarelo      │
   │ HC ≤ 7%         │ OK      │ Verde        │
   ├─────────────────┼─────────┼──────────────┤
   │ Total > 7%      │ FAIL    │ Vermelho     │
   │ Total > 5%      │ ALERT   │ Amarelo      │
   │ Total ≤ 5%      │ OK      │ Verde        │
   └─────────────────┴─────────┴──────────────┘
   
   Ação:
   • FAIL → INVESTIGAR
   • ALERT/OK → MONITORAR
   ↓

7. PERSISTÊNCIA
---------------
   Frontend:
     useAppStore.importMonitoringData(data)
     → Zustand State
     → localStorage (auto-save)
   
   Backend:
     POST /api/measurements/daily
     → SQLite: daily_measurement table
     → Retorna: { success, records_inserted }
